# Reusable design brief template

Use this exact structure unless the user asks for a different format.

## Project frame
- Brand / working title:
- Source type: business idea / current client site / competitor / inspiration / mixed
- Primary website goal:
- Main CTA:
- Target buyers:
- Default languages: Arabic + English
- Delivery mode: staged approvals

## Business summary
Write a short paragraph that explains the business, what it sells, and who it serves.

## Premium repositioning
State how the new website should feel more expensive and more credible than the source or current market norm.

## Trust strategy
List the signals the website should emphasize.
Examples:
- sourcing strength
- years of experience
- markets served
- certifications
- logistics/process clarity
- proof of delivery or outcomes
- client categories

## Visual direction
- style keywords:
- overall mood:
- palette direction:
- typography direction:
- imagery direction:
- material cues:
- what to avoid:

## Layout personality
Describe the page composition style.
Examples: editorial blocks, cinematic hero, calm whitespace, premium grids, large section transitions.

## Motion direction
Describe the motion character in words first, then name the main interaction patterns.
Keep it premium, strong, and restrained.

## Page scope
List the pages to generate.
Default:
- home
- about
- products
- services
- contact

## Section priorities
List the sections that should do the most persuasive work.
Prioritize work, process, and trust when relevant.

## Bilingual UX notes
Describe:
- AR/EN toggle behavior
- rtl/ltr behavior
- copy length risks
- typography balancing across languages
- icon and directional treatment

## Implementation notes
- stack: vite + react + tailwind
- deployment target: vercel
- components should be reusable
- content should be structured for future translation files
- animation should stay migration-aware for later tall implementation

## Approval question
End with one direct approval prompt.
